The EmberHarmony CLI by default starts the [TUI](/docs/tui) when run without any arguments.

```bash
emberharmony
```

But it also accepts commands as documented on this page. This allows you to interact with EmberHarmony programmatically.

```bash
emberharmony run "Explain how closures work in JavaScript"
```

---

### tui

Start the EmberHarmony terminal user interface.

```bash
emberharmony [project]
```

#### Flags

| Flag         | Short | Description                                |
| ------------ | ----- | ------------------------------------------ |
| `--continue` | `-c`  | Continue the last session                  |
| `--session`  | `-s`  | Session ID to continue                     |
| `--prompt`   |       | Prompt to use                              |
| `--model`    | `-m`  | Model to use in the form of provider/model |
| `--agent`    |       | Agent to use                               |
| `--port`     |       | Port to listen on                          |
| `--hostname` |       | Hostname to listen on                      |

## Commands

The EmberHarmony CLI also has the following commands.

---

### agent

Manage agents for EmberHarmony.

```bash
emberharmony agent [command]
```

---

### attach

Attach a terminal to an already running EmberHarmony backend server started via `serve` or `web` commands.

```bash
emberharmony attach [url]
```

This allows using the TUI with a remote EmberHarmony backend. For example:

```bash
# Start the backend server for web/mobile access
emberharmony web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
emberharmony attach http://10.20.30.40:4096
```

#### Flags

| Flag        | Short | Description                       |
| ----------- | ----- | --------------------------------- |
| `--dir`     |       | Working directory to start TUI in |
| `--session` | `-s`  | Session ID to continue            |

---

#### create

Create a new agent with custom configuration.

```bash
emberharmony agent create
```

This command will guide you through creating a new agent with a custom system prompt and tool configuration.

---

#### list

List all available agents.

```bash
emberharmony agent list
```

---

### auth

Command to manage credentials and login for providers.

```bash
emberharmony auth [command]
```

---

#### login

EmberHarmony is powered by the provider list at [Models.dev](https://models.dev), so you can use `emberharmony auth login` to configure API keys for any provider you'd like to use. This is stored in `~/.local/share/emberharmony/auth.json`.

```bash
emberharmony auth login
```

When EmberHarmony starts up it loads the providers from the credentials file. And if there are any keys defined in your environments or a `.env` file in your project.

---

#### list

Lists all the authenticated providers as stored in the credentials file.

```bash
emberharmony auth list
```

Or the short version.

```bash
emberharmony auth ls
```

---

#### logout

Logs you out of a provider by clearing it from the credentials file.

```bash
emberharmony auth logout
```

---

### github

Manage the GitHub agent for repository automation.

```bash
emberharmony github [command]
```

---

#### install

Install the GitHub agent in your repository.

```bash
emberharmony github install
```

This sets up the necessary GitHub Actions workflow and guides you through the configuration process. [Learn more](/docs/github).

---

#### run

Run the GitHub agent. This is typically used in GitHub Actions.

```bash
emberharmony github run
```

##### Flags

| Flag      | Description                            |
| --------- | -------------------------------------- |
| `--event` | GitHub mock event to run the agent for |
| `--token` | GitHub personal access token           |

### mcp

Manage Model Context Protocol servers.

```bash
emberharmony mcp [command]
```

---

#### add

Add an MCP server to your configuration.

```bash
emberharmony mcp add
```

This command will guide you through adding either a local or remote MCP server.

---

#### list

List all configured MCP servers and their connection status.

```bash
emberharmony mcp list
```

Or use the short version.

```bash
emberharmony mcp ls
```

---

#### auth

Authenticate with an OAuth-enabled MCP server.

```bash
emberharmony mcp auth [name]
```

If you don't provide a server name, you'll be prompted to select from available OAuth-capable servers.

You can also list OAuth-capable servers and their authentication status.

```bash
emberharmony mcp auth list
```

Or use the short version.

```bash
emberharmony mcp auth ls
```

---

#### logout

Remove OAuth credentials for an MCP server.

```bash
emberharmony mcp logout [name]
```

---

#### debug

Debug OAuth connection issues for an MCP server.

```bash
emberharmony mcp debug <name>
```

### models

List all available models from configured providers.

```bash
emberharmony models [provider]
```

This command displays all models available across your configured providers in the format `provider/model`.

This is useful for figuring out the exact model name to use in [your config](/docs/config/).

You can optionally pass a provider ID to filter models by that provider.

```bash
emberharmony models anthropic
```

#### Flags

| Flag        | Description                                                  |
| ----------- | ------------------------------------------------------------ |
| `--refresh` | Refresh the models cache from models.dev                     |
| `--verbose` | Use more verbose model output (includes metadata like costs) |

Use the `--refresh` flag to update the cached model list. This is useful when new models have been added to a provider and you want to see them in EmberHarmony.

```bash
emberharmony models --refresh
```

### run

Run emberharmony in non-interactive mode by passing a prompt directly.

```bash
emberharmony run [message..]
```

This is useful for scripting, automation, or when you want a quick answer without launching the full TUI. For example.

```bash "emberharmony run"
emberharmony run Explain the use of context in Go
```

You can also attach to a running `emberharmony serve` instance to avoid MCP server cold boot times on every run:

```bash
# Start a headless server in one terminal
emberharmony serve

# In another terminal, run commands that attach to it
emberharmony run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Flags

| Flag         | Short | Description                                                           |
| ------------ | ----- | --------------------------------------------------------------------- |
| `--command`  |       | The command to run, use message for args                              |
| `--continue` | `-c`  | Continue the last session                                             |
| `--session`  | `-s`  | Session ID to continue                                                |
| `--share`    |       | Share the session                                                     |
| `--model`    | `-m`  | Model to use in the form of provider/model                            |
| `--agent`    |       | Agent to use                                                          |
| `--file`     | `-f`  | File(s) to attach to message                                          |
| `--format`   |       | Format: default (formatted) or json (raw JSON events)                 |
| `--title`    |       | Title for the session (uses truncated prompt if no value provided)    |
| `--attach`   |       | Attach to a running emberharmony server (e.g., http://localhost:4096) |
| `--port`     |       | Port for the local server (defaults to random port)                   |

### serve

Start a headless EmberHarmony server for API access. Check out the [server docs](/docs/server) for the full HTTP interface.

```bash
emberharmony serve
```

This starts an HTTP server that provides API access to emberharmony functionality without the TUI interface. Set `EMBERHARMONY_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `emberharmony`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

### session

Manage EmberHarmony sessions.

```bash
emberharmony session [command]
```

---

#### list

List all EmberHarmony sessions.

```bash
emberharmony session list
```

##### Flags

| Flag          | Short | Description                          |
| ------------- | ----- | ------------------------------------ |
| `--max-count` | `-n`  | Limit to N most recent sessions      |
| `--format`    |       | Output format: table or json (table) |

---

### stats

Show token usage and cost statistics for your EmberHarmony sessions.

```bash
emberharmony stats
```

#### Flags

| Flag        | Description                                                                 |
| ----------- | --------------------------------------------------------------------------- |
| `--days`    | Show stats for the last N days (all time)                                   |
| `--tools`   | Number of tools to show (all)                                               |
| `--models`  | Show model usage breakdown (hidden by default). Pass a number to show top N |
| `--project` | Filter by project (all projects, empty string: current project)             |

---

### export

Export session data as JSON.

```bash
emberharmony export [sessionID]
```

If you don't provide a session ID, you'll be prompted to select from available sessions.

---

### import

Import session data from a JSON file or EmberHarmony share URL.

```bash
emberharmony import <file>
```

You can import from a local file or an EmberHarmony share URL.

```bash
emberharmony import session.json
emberharmony import https://solace.ofharmony.ai/s/abc123
```

---

### web

Start a headless EmberHarmony server with a web interface.

```bash
emberharmony web
```

This starts an HTTP server and opens a web browser to access EmberHarmony through a web interface. Set `EMBERHARMONY_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `emberharmony`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

---

### acp

Start an ACP (Agent Client Protocol) server.

```bash
emberharmony acp
```

This command starts an ACP server that communicates via stdin/stdout using nd-JSON.

#### Flags

| Flag         | Description           |
| ------------ | --------------------- |
| `--cwd`      | Working directory     |
| `--port`     | Port to listen on     |
| `--hostname` | Hostname to listen on |

---

### uninstall

Uninstall EmberHarmony and remove all related files.

```bash
emberharmony uninstall
```

#### Flags

| Flag            | Short | Description                                 |
| --------------- | ----- | ------------------------------------------- |
| `--keep-config` | `-c`  | Keep configuration files                    |
| `--keep-data`   | `-d`  | Keep session data and snapshots             |
| `--dry-run`     |       | Show what would be removed without removing |
| `--force`       | `-f`  | Skip confirmation prompts                   |

---

### upgrade

Updates emberharmony to the latest version or a specific version.

```bash
emberharmony upgrade [target]
```

To upgrade to the latest version.

```bash
emberharmony upgrade
```

To upgrade to a specific version.

```bash
emberharmony upgrade v0.1.48
```

#### Flags

| Flag       | Short | Description                                                 |
| ---------- | ----- | ----------------------------------------------------------- |
| `--method` | `-m`  | The installation method that was used; curl, npm, pnpm, bun |

## Environment variables

EmberHarmony can be configured using environment variables.

| Variable                              | Type    | Description                                           |
| ------------------------------------- | ------- | ----------------------------------------------------- |
| `EMBERHARMONY_AUTO_SHARE`                 | boolean | Automatically share sessions                          |
| `EMBERHARMONY_GIT_BASH_PATH`              | string  | Path to Git Bash executable on Windows                |
| `EMBERHARMONY_CONFIG`                     | string  | Path to config file                                   |
| `EMBERHARMONY_CONFIG_DIR`                 | string  | Path to config directory                              |
| `EMBERHARMONY_CONFIG_CONTENT`             | string  | Inline json config content                            |
| `EMBERHARMONY_DISABLE_AUTOUPDATE`         | boolean | Disable automatic update checks                       |
| `EMBERHARMONY_DISABLE_PRUNE`              | boolean | Disable pruning of old data                           |
| `EMBERHARMONY_DISABLE_TERMINAL_TITLE`     | boolean | Disable automatic terminal title updates              |
| `EMBERHARMONY_PERMISSION`                 | string  | Inlined json permissions config                       |
| `EMBERHARMONY_DISABLE_DEFAULT_PLUGINS`    | boolean | Disable default plugins                               |
| `EMBERHARMONY_DISABLE_LSP_DOWNLOAD`       | boolean | Disable automatic LSP server downloads                |
| `EMBERHARMONY_ENABLE_EXPERIMENTAL_MODELS` | boolean | Enable experimental models                            |
| `EMBERHARMONY_DISABLE_AUTOCOMPACT`        | boolean | Disable automatic context compaction                  |
| `EMBERHARMONY_DISABLE_CLAUDE_CODE`        | boolean | Disable reading from `.claude` (prompt + skills)      |
| `EMBERHARMONY_DISABLE_CLAUDE_CODE_PROMPT` | boolean | Disable reading `~/.claude/CLAUDE.md`                 |
| `EMBERHARMONY_DISABLE_CLAUDE_CODE_SKILLS` | boolean | Disable loading `.claude/skills`                      |
| `EMBERHARMONY_CLIENT`                     | string  | Client identifier (defaults to `cli`)                 |
| `EMBERHARMONY_ENABLE_EXA`                 | boolean | Enable Exa web search tools                           |
| `EMBERHARMONY_SERVER_PASSWORD`            | string  | Enable basic auth for `serve`/`web`                   |
| `EMBERHARMONY_SERVER_USERNAME`            | string  | Override basic auth username (default `emberharmony`) |
