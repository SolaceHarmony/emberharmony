The CodeHarmony CLI by default starts the [TUI](/docs/tui) when run without any arguments.

```bash
code-harmony
```

But it also accepts commands as documented on this page. This allows you to interact with CodeHarmony programmatically.

```bash
code-harmony run "Explain how closures work in JavaScript"
```

---

### tui

Start the CodeHarmony terminal user interface.

```bash
code-harmony [project]
```

#### Flags

| Flag         | Short | Description                                |
| ------------ | ----- | ------------------------------------------ |
| `--continue` | `-c`  | Continue the last session                  |
| `--session`  | `-s`  | Session ID to continue                     |
| `--prompt`   |       | Prompt to use                              |
| `--model`    | `-m`  | Model to use in the form of provider/model |
| `--agent`    |       | Agent to use                               |
| `--port`     |       | Port to listen on                          |
| `--hostname` |       | Hostname to listen on                      |

## Commands

The CodeHarmony CLI also has the following commands.

---

### agent

Manage agents for CodeHarmony.

```bash
code-harmony agent [command]
```

---

### attach

Attach a terminal to an already running CodeHarmony backend server started via `serve` or `web` commands.

```bash
code-harmony attach [url]
```

This allows using the TUI with a remote CodeHarmony backend. For example:

```bash
# Start the backend server for web/mobile access
code-harmony web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
code-harmony attach http://10.20.30.40:4096
```

#### Flags

| Flag        | Short | Description                       |
| ----------- | ----- | --------------------------------- |
| `--dir`     |       | Working directory to start TUI in |
| `--session` | `-s`  | Session ID to continue            |

---

#### create

Create a new agent with custom configuration.

```bash
code-harmony agent create
```

This command will guide you through creating a new agent with a custom system prompt and tool configuration.

---

#### list

List all available agents.

```bash
code-harmony agent list
```

---

### auth

Command to manage credentials and login for providers.

```bash
code-harmony auth [command]
```

---

#### login

CodeHarmony is powered by the provider list at [Models.dev](https://models.dev), so you can use `code-harmony auth login` to configure API keys for any provider you'd like to use. This is stored in `~/.local/share/code-harmony/auth.json`.

```bash
code-harmony auth login
```

When CodeHarmony starts up it loads the providers from the credentials file. And if there are any keys defined in your environments or a `.env` file in your project.

---

#### list

Lists all the authenticated providers as stored in the credentials file.

```bash
code-harmony auth list
```

Or the short version.

```bash
code-harmony auth ls
```

---

#### logout

Logs you out of a provider by clearing it from the credentials file.

```bash
code-harmony auth logout
```

---

### github

Manage the GitHub agent for repository automation.

```bash
code-harmony github [command]
```

---

#### install

Install the GitHub agent in your repository.

```bash
code-harmony github install
```

This sets up the necessary GitHub Actions workflow and guides you through the configuration process. [Learn more](/docs/github).

---

#### run

Run the GitHub agent. This is typically used in GitHub Actions.

```bash
code-harmony github run
```

##### Flags

| Flag      | Description                            |
| --------- | -------------------------------------- |
| `--event` | GitHub mock event to run the agent for |
| `--token` | GitHub personal access token           |

### mcp

Manage Model Context Protocol servers.

```bash
code-harmony mcp [command]
```

---

#### add

Add an MCP server to your configuration.

```bash
code-harmony mcp add
```

This command will guide you through adding either a local or remote MCP server.

---

#### list

List all configured MCP servers and their connection status.

```bash
code-harmony mcp list
```

Or use the short version.

```bash
code-harmony mcp ls
```

---

#### auth

Authenticate with an OAuth-enabled MCP server.

```bash
code-harmony mcp auth [name]
```

If you don't provide a server name, you'll be prompted to select from available OAuth-capable servers.

You can also list OAuth-capable servers and their authentication status.

```bash
code-harmony mcp auth list
```

Or use the short version.

```bash
code-harmony mcp auth ls
```

---

#### logout

Remove OAuth credentials for an MCP server.

```bash
code-harmony mcp logout [name]
```

---

#### debug

Debug OAuth connection issues for an MCP server.

```bash
code-harmony mcp debug <name>
```

### models

List all available models from configured providers.

```bash
code-harmony models [provider]
```

This command displays all models available across your configured providers in the format `provider/model`.

This is useful for figuring out the exact model name to use in [your config](/docs/config/).

You can optionally pass a provider ID to filter models by that provider.

```bash
code-harmony models anthropic
```

#### Flags

| Flag        | Description                                                  |
| ----------- | ------------------------------------------------------------ |
| `--refresh` | Refresh the models cache from models.dev                     |
| `--verbose` | Use more verbose model output (includes metadata like costs) |

Use the `--refresh` flag to update the cached model list. This is useful when new models have been added to a provider and you want to see them in CodeHarmony.

```bash
code-harmony models --refresh
```

### run

Run code-harmony in non-interactive mode by passing a prompt directly.

```bash
code-harmony run [message..]
```

This is useful for scripting, automation, or when you want a quick answer without launching the full TUI. For example.

```bash "code-harmony run"
code-harmony run Explain the use of context in Go
```

You can also attach to a running `code-harmony serve` instance to avoid MCP server cold boot times on every run:

```bash
# Start a headless server in one terminal
code-harmony serve

# In another terminal, run commands that attach to it
code-harmony run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Flags

| Flag         | Short | Description                                                           |
| ------------ | ----- | --------------------------------------------------------------------- |
| `--command`  |       | The command to run, use message for args                              |
| `--continue` | `-c`  | Continue the last session                                             |
| `--session`  | `-s`  | Session ID to continue                                                |
| `--share`    |       | Share the session                                                     |
| `--model`    | `-m`  | Model to use in the form of provider/model                            |
| `--agent`    |       | Agent to use                                                          |
| `--file`     | `-f`  | File(s) to attach to message                                          |
| `--format`   |       | Format: default (formatted) or json (raw JSON events)                 |
| `--title`    |       | Title for the session (uses truncated prompt if no value provided)    |
| `--attach`   |       | Attach to a running code-harmony server (e.g., http://localhost:4096) |
| `--port`     |       | Port for the local server (defaults to random port)                   |

### serve

Start a headless CodeHarmony server for API access. Check out the [server docs](/docs/server) for the full HTTP interface.

```bash
code-harmony serve
```

This starts an HTTP server that provides API access to code-harmony functionality without the TUI interface. Set `CODE_HARMONY_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `code-harmony`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

```bash
code-harmony session [command]
```

---

#### list

List all CodeHarmony sessions.

```bash
code-harmony session list
```

##### Flags

| Flag          | Short | Description                          |
| ------------- | ----- | ------------------------------------ |
| `--max-count` | `-n`  | Limit to N most recent sessions      |
| `--format`    |       | Output format: table or json (table) |

---

### stats

Show token usage and cost statistics for your CodeHarmony sessions.

```bash
code-harmony stats
```

#### Flags

| Flag        | Description                                                                 |
| ----------- | --------------------------------------------------------------------------- |
| `--days`    | Show stats for the last N days (all time)                                   |
| `--tools`   | Number of tools to show (all)                                               |
| `--models`  | Show model usage breakdown (hidden by default). Pass a number to show top N |
| `--project` | Filter by project (all projects, empty string: current project)             |

---

### export

Export session data as JSON.

```bash
code-harmony export [sessionID]
```

If you don't provide a session ID, you'll be prompted to select from available sessions.

---

### import

Import session data from a JSON file or CodeHarmony share URL.

```bash
code-harmony import <file>
```

You can import from a local file or an CodeHarmony share URL.

```bash
code-harmony import session.json
code-harmony import https://solace.ofharmony.ai/s/abc123
```

---

### web

Start a headless CodeHarmony server with a web interface.

```bash
code-harmony web
```

This starts an HTTP server and opens a web browser to access CodeHarmony through a web interface. Set `CODE_HARMONY_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `code-harmony`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

---

### acp

Start an ACP (Agent Client Protocol) server.

```bash
code-harmony acp
```

This command starts an ACP server that communicates via stdin/stdout using nd-JSON.

#### Flags

| Flag         | Description           |
| ------------ | --------------------- |
| `--cwd`      | Working directory     |
| `--port`     | Port to listen on     |
| `--hostname` | Hostname to listen on |

---

### uninstall

Uninstall CodeHarmony and remove all related files.

```bash
code-harmony uninstall
```

#### Flags

| Flag            | Short | Description                                 |
| --------------- | ----- | ------------------------------------------- |
| `--keep-config` | `-c`  | Keep configuration files                    |
| `--keep-data`   | `-d`  | Keep session data and snapshots             |
| `--dry-run`     |       | Show what would be removed without removing |
| `--force`       | `-f`  | Skip confirmation prompts                   |

---

### upgrade

Updates code-harmony to the latest version or a specific version.

```bash
code-harmony upgrade [target]
```

To upgrade to the latest version.

```bash
code-harmony upgrade
```

To upgrade to a specific version.

```bash
code-harmony upgrade v0.1.48
```

#### Flags

| Flag       | Short | Description                                                 |
| ---------- | ----- | ----------------------------------------------------------- |
| `--method` | `-m`  | The installation method that was used; curl, npm, pnpm, bun |

## Environment variables

CodeHarmony can be configured using environment variables.

| Variable                              | Type    | Description                                           |
| ------------------------------------- | ------- | ----------------------------------------------------- |
| `CODE_HARMONY_AUTO_SHARE`                 | boolean | Automatically share sessions                          |
| `CODE_HARMONY_GIT_BASH_PATH`              | string  | Path to Git Bash executable on Windows                |
| `CODE_HARMONY_CONFIG`                     | string  | Path to config file                                   |
| `CODE_HARMONY_CONFIG_DIR`                 | string  | Path to config directory                              |
| `CODE_HARMONY_CONFIG_CONTENT`             | string  | Inline json config content                            |
| `CODE_HARMONY_DISABLE_AUTOUPDATE`         | boolean | Disable automatic update checks                       |
| `CODE_HARMONY_DISABLE_PRUNE`              | boolean | Disable pruning of old data                           |
| `CODE_HARMONY_DISABLE_TERMINAL_TITLE`     | boolean | Disable automatic terminal title updates              |
| `CODE_HARMONY_PERMISSION`                 | string  | Inlined json permissions config                       |
| `CODE_HARMONY_DISABLE_DEFAULT_PLUGINS`    | boolean | Disable default plugins                               |
| `CODE_HARMONY_DISABLE_LSP_DOWNLOAD`       | boolean | Disable automatic LSP server downloads                |
| `CODE_HARMONY_ENABLE_EXPERIMENTAL_MODELS` | boolean | Enable experimental models                            |
| `CODE_HARMONY_DISABLE_AUTOCOMPACT`        | boolean | Disable automatic context compaction                  |
| `CODE_HARMONY_DISABLE_CLAUDE_CODE`        | boolean | Disable reading from `.claude` (prompt + skills)      |
| `CODE_HARMONY_DISABLE_CLAUDE_CODE_PROMPT` | boolean | Disable reading `~/.claude/CLAUDE.md`                 |
| `CODE_HARMONY_DISABLE_CLAUDE_CODE_SKILLS` | boolean | Disable loading `.claude/skills`                      |
| `CODE_HARMONY_CLIENT`                     | string  | Client identifier (defaults to `cli`)                 |
| `CODE_HARMONY_ENABLE_EXA`                 | boolean | Enable Exa web search tools                           |
| `CODE_HARMONY_SERVER_PASSWORD`            | string  | Enable basic auth for `serve`/`web`                   |
| `CODE_HARMONY_SERVER_USERNAME`            | string  | Override basic auth username (default `code-harmony`) |
